# Migrating from Pelican
This guide will show you how to migrate from your old Pelican Panel, to your new Calagopus Panel.

## What will be migrated?
Basically everything **EXCEPT** API keys generated by Pelican, due to Calagopus using a different format than Pelican, both not reversible.

## Prerequisites
To migrate from Pelican, you must have:
* Your Dockerized Pelican `.env` file ready.
* Calagopus Panel installed, but not fully configured (stop at the OOBE)

## Installation
If you haven't already so, install Calagopus Panel first using [this guide](../../panel/installation.md). Once you have installed Calagopus Panel, and arrived to the OOBE, continue following this guide.

::: warning DO NOT PROGRESS THROUGH THE OOBE!
This guide requires a fresh database because we need to replace it with our old data. If you finished the OOBE, you will need to delete the database. Once you arrived to the following image below, close the tab and continue following this tab.
![Calagopus Panel OOBE](../../panel/oobe.png)

::: details How to delete the database?
::::tabs
=== Docker
Head to the directory where the compose file is, then stop Calagopus:
```bash
docker compose down
```
Then, nuke the database:
```bash
# We're nuking the database because we need to replace it with our old data
rm -r postgres
```
Finally, start Calagopus again:
```bash
docker compose up
```
=== APT/RPM, Binary
Shut down Calagopus first:
```bash
# Linux
systemctl stop calagopus-panel

# MacOS
# soon

# Windows
nssm stop "Calagopus Panel"
```

Then, login to the Postgres database:
```bash
# If on Linux or MacOS
sudo -u postgres psql

# If on Windows, follow the steps in https://calagopus.com/docs/panel/installation/binary#database-configuration
```
Then, nuke the database and remake it:
```sql
DROP DATABASE panel (FORCE);
CREATE DATABASE panel OWNER calagopus;
GRANT ALL PRIVILEGES ON DATABASE panel TO calagopus;
quit
```
Finally, start Calagopus back:
```bash
# Linux
systemctl start calagopus-panel

# MacOS
# soon

# Windows
nssm start "Calagopus Panel"
```
:::

First, define your Pelican directory using the `PELICAN_DIRECTORY` variable. This is not needed for this guide, but useful for this guide. If your Pelican host mount isn't located at `/srv/pelican`, change the path of the command below.
```bash
# Use this command to set a $PELICAN_DIRECTORY variable
# for use later in this guide.
export PELICAN_DIRECTORY=/srv/pelican
```

=== Docker
### Making the .env file
::: warning
This step requires you to navigate to your Dockerized Pelican folder where it should contain your `docker-compose.yml` file and/or your `.env` file.
:::

To import data from Pelican, you need `APP_URL`, `APP_KEY`, and the correct database variables from your Pelican `.env` file.

Pelican supports these source database drivers:
* `mysql`
* `mariadb`
* `sqlite`
* `sqlite3`

Open a notepad where you will be storing the variables, you will be needing it later to actually create the `.env` file.

#### APP_URL
The `APP_URL` is your existing Pelican domain, setup either in the `docker-compose.yml` file, or in the `.env` file of Pelican.
An example value could be:
```env
APP_URL=https://panel.local.test
```

#### APP_KEY
To find `APP_KEY`, run the following command:
```bash
cat $PELICAN_DIRECTORY/.env
```
Copy the output that the command gave you and paste it to your notepad.
An example value could be:
```env
APP_KEY=base64:xc5QXq4u3Qgi3zRP0Q9qq32mnZvl0lVY
```

#### DB_CONNECTION
To find your `DB_CONNECTION`, look at Pelican's `.env` file. An example value could be:
```env
DB_CONNECTION=mysql
```

#### MySQL / MariaDB
If `DB_CONNECTION` is `mysql` or `mariadb`, you need these variables too:
* `DB_HOST`
* `DB_PORT`
* `DB_DATABASE`
* `DB_USERNAME`
* `DB_PASSWORD`

If your database container is on the same Docker network, you can find the container IP using:
```bash
# On Linux/MacOS
echo "DB_HOST=$(docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' $(docker compose ps -q database))"

# On Windows, instead run the command below:
echo "DB_HOST=$($(docker compose ps -q database) | foreach { docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' $_ })"
```

An example MySQL/MariaDB block could be:
```env
DB_CONNECTION=mysql
DB_HOST=172.29.0.4
DB_PORT=3306
DB_DATABASE=pelican
DB_USERNAME=pelican
DB_PASSWORD=secret
```

#### SQLite3
If `DB_CONNECTION` is `sqlite` or `sqlite3`, you only need `DB_DATABASE`.
An example value could be:
```env
DB_CONNECTION=sqlite
DB_DATABASE=/database.sqlite
```

If your Pelican `.env` originally uses a relative path like `database/database.sqlite`, you can still import successfully in Docker by copying the database into the container root and changing the copied `.env` file so `DB_DATABASE=/database.sqlite`.

Once you have the environment variables on your notepad, head to the Calagopus directory with the `compose.yml` file, and create a new file using your favorite text editor named `pelican.env`, where you can paste the contents of the notepad.

### Importing data
Once your `pelican.env` file has been created, check if it exists, then run this command:
```bash
docker compose cp pelican.env web:/.env
```

If Pelican uses SQLite3, also copy the SQLite database file into the container root:
```bash
docker compose cp $PELICAN_DIRECTORY/database/database.sqlite web:/database.sqlite
```

Then, simply run the migration command:
```bash
docker compose exec web panel-rs import pelican --environment /.env
```

This command will start migrating all of your users, servers, nodes, etc, which may take a while. Once it's done migrating, simply restart the container:
```bash
docker compose down
docker compose up -d
```

Login with your existing Pelican credentials, and you should be all set!
