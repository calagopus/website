# Migrating from Pelican
This guide shows how to migrate from your old standalone Pelican Panel to your new Calagopus Panel.

## What will be migrated?
Basically everything **EXCEPT** API keys generated by Pelican, because Calagopus uses a different format and they cannot be converted safely.

## Prerequisites
To migrate from Pelican, you must have:
* Your Pelican `.env` file ready
* Calagopus Panel installed, but not fully configured yet
* Access to the Pelican database source, which can be either MySQL/MariaDB or SQLite3

## Installation
If you have not already done so, install Calagopus Panel first using [this guide](../../panel/installation.md). Once you have installed Calagopus Panel and arrived at the OOBE, continue following this guide.

::: warning DO NOT PROGRESS THROUGH THE OOBE!
This guide assumes a fresh Calagopus database because we need to replace it with your Pelican data. If you already completed the OOBE, delete and recreate the database first.
:::

## Supported Pelican databases
The Pelican importer supports these source database backends:

* `mysql`
* `mariadb`
* `sqlite`
* `sqlite3`

The importer reads the source connection details directly from the Pelican `.env` file you pass with `--environment`.

### MySQL / MariaDB
Your Pelican `.env` should contain values like:

```env
APP_URL=https://panel.example.com
APP_KEY=base64:...
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=pelican
DB_USERNAME=pelican
DB_PASSWORD=secret
```

### SQLite3
Your Pelican `.env` should contain values like:

```env
APP_URL=https://panel.example.com
APP_KEY=base64:...
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite
```

If `DB_DATABASE` is a relative path, it is resolved relative to the Pelican `.env` file location.

## Importing data
Run the importer from the machine where Calagopus is installed.

If your Pelican installation is at `/var/www/pelican`, run:

```bash
calagopus-panel import pelican --environment /var/www/pelican/.env
```

If Pelican uses SQLite3 and `DB_DATABASE` is a relative path, nothing extra is needed as long as the SQLite file still exists relative to that `.env` file.

Once the import finishes, restart Calagopus:

```bash
# Linux
systemctl restart calagopus-panel

# Windows
nssm restart "Calagopus Panel"
```

Login with your existing Pelican credentials afterwards and verify your users, servers, nodes, and eggs were migrated successfully.
