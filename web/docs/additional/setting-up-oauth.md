# Setting up OAuth

OAuth allows users to sign in to the panel using an existing identity provider instead of a panel-specific password. This reduces friction for onboarding, eliminates panel-specific password reset flows, and lets communities centralize identity with a provider they already manage.

Supported providers:

::::tabs
=== GitHub
See the [GitHub OAuth](./setting-up-oauth/github.md) guide.
=== Google
See the [Google OAuth](./setting-up-oauth/google.md) guide.
=== Discord
See the [Discord OAuth](./setting-up-oauth/discord.md) guide.
=== Generic (OIDC)
See the [Generic OAuth](./setting-up-oauth/generic.md) guide for any OpenID Connect compatible provider.
::::
